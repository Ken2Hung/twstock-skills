# twstock-backtesting

## 1. 觸發時機

當使用者想知道某策略在**歷史**上的表現時觸發，例如：
- 「**回測** 2330 均線多頭策略 2022 到 2024」
- 「幫我**回測**台積電這三年，看報酬和**最大回撤**」
- 「**扣掉手續費和證交稅**後這策略還賺嗎」

即時選股屬 `twstock-screening-stocks`；持股健檢屬 `twstock-reviewing-portfolio`。

## 2. 能力總覽（編排流程）

L3 **編排型**：

```
解析回測參數（證券/期間/策略）→ 委派引擎（經 twstock-module 取歷史日K）→ 回測報告
```

**分層鐵律（紅線 #2）**：本場景與引擎 SHALL NOT 直接呼叫資料源 API；歷史資料一律經 `twstock-module`。

## 3. 輸入

- 證券代碼、期間（起訖）、策略訊號（v1.0 附「均線多頭 5>20>60」參考訊號，可插拔）。

## 4. 委派引擎

呼叫 [`scripts/backtest.py`](scripts/backtest.py)：`--stock-id --start --end`。引擎 import `twstock-module` 取歷史日K（不直接碰 API），逐日模擬 long-only、套用台股成本、回傳指標。

## 5. 回測方法（引擎保證）

- **long-only、close-to-close**；`position = signal.shift(1)` → **無 look-ahead bias**（決策資料嚴格早於報酬期）。
- **台股交易成本**：買 0.1425%（低消 20 元）、賣 0.4425%（手續費低消 20 + 證交稅 0.3%），引用 `tw-market-rules.md`。
- 資料不足 60 筆（60MA 暖身）或有 data_gaps → **fail-open**，回報無法回測、不腦補。

## 6. 回測報告輸出

固定呈現：

| 欄位 | 內容 |
|---|---|
| 期間 / 訊號 | 回測區間與策略 |
| 總報酬（淨 / 毛） | 扣成本後 vs 扣成本前 |
| **成本侵蝕** | 毛 − 淨（台股成本吃掉多少） |
| 年化報酬 | 淨報酬年化 |
| 最大回撤 (MDD) | 權益曲線最大回落 |
| 勝率 / 交易次數 | 獲利平倉比 / 來回次數 |
| data_gaps | 資料缺口 |

## 7. 範例（回測報告）

```
【2330 台積電｜均線多頭 5>20>60｜2022-01-01 ~ 2024-12-31】
總報酬：淨 +8.23%（毛 +19.57%）
成本侵蝕：-11.34%  ← 17 筆交易 × 台股來回成本
年化報酬：+2.78%    最大回撤：-17.89%
勝率：47%（17 筆）
data_gaps：[]
⚠️ 免責：回測為歷史模擬，過去績效不代表未來；不構成投資建議，請自行判斷並自負風險。
```

## 8. 合規（紅線）

- **data_gaps 非空** → 報告 MUST 明示「回測基於不完整資料」。
- 報告必附**免責聲明**；**禁止**「保證」「必賺」「穩賺」等表述。
- 回測損益**一律扣台股交易成本**，成本參數引用 `tw-market-rules.md`、不硬編碼。

## 9. 限制（v1.0 範圍）

- **單檔 long-only** MVP；多檔投組回測、完整三面向評分回測、參數優化/walk-forward、放空 = future。
- close-to-close 模型；未計滑價與流動性衝擊（處置股/漲跌停未成交風險）——報告應提醒。

## 10. 使用方式

以自然語言提供「證券 + 期間 + 策略」即可，例如「回測 2330 均線多頭 2022 到 2024」。場景委派引擎、輸出上述報告。

## 11. 分層邊界與免責

- 本場景為 **L3 企業場景**：編排 + 報告；歷史資料一律委派 L2 `twstock-module`。
- 回測為歷史模擬，**過去績效不代表未來**，所有輸出不構成投資建議；台股有漲跌停與流動性風險，投資決策請自行判斷並諮詢專業人士。
