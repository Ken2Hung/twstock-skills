# twstock-screening-stocks

## 1. 觸發時機

當使用者以自然語言表達**選股／篩選**需求時觸發，例如：
- 「**幫我篩選**外資連續買超 5 天、月營收年增 > 20% 的上市電子股」
- 「**找出**站上 60 日均線且融資減少的**股票**」
- 「**用**高股息**策略挑** 5 檔金融股，附目標價與止損價」

單檔純資料查詢（如「給我 2330 的日K」）屬 `twstock-module` 職責，不走本場景。

## 2. 能力總覽（編排流程）

本場景為 **L3 編排型**，四階段：

```
解析自然語言條件 → 委派 twstock-module 取得資料 → 套用篩選 → 輸出決策看板
```

**分層鐵律（紅線 #2）**：本場景 **SHALL NOT** 直接呼叫 FinMind／yfinance／TWSE／TPEx 或任何資料源 API、亦不自行撰寫取數邏輯；**凡需任何台股資料，一律委派 `twstock-module`**。

## 3. 輸入：自然語言條件解析

把使用者條件解析為三類篩選面向，對應委派 `twstock-module` 的 dataset：

| 面向 | 常見條件 | 委派 dataset |
|---|---|---|
| 基本面 | 月營收 YoY、EPS、ROE | `revenue`、`financial` |
| 技術面 | 均線、乖離、量價 | `daily`、`per_pbr` |
| 籌碼面 | 三大法人、融資融券 | `institutional`、`margin` |

（dataset 用法見 `references/finmind-api-cheatsheet.md`。）**須待所有委派回傳結構化 JSON 後才進入篩選。**

## 4. 委派取數（禁令）

- 每一項資料需求 → 呼叫 `twstock-module`（capability `twstock-data-fetching`）。
- **禁止**在本場景直接 import FinMind/yfinance、發 HTTP、或讀交易所開放資料。
- `twstock-module` 回傳含 `source`／`fetched_at`／`data_gaps`；本場景**原樣尊重 data_gaps**，不補值。

## 5. 篩選

對委派回傳的結構化資料套用使用者條件（門檻由使用者語意決定；系統化指標定義見 `references/scoring-model.md`）。通過者進入評分與看板。

## 6. 決策看板輸出契約（固定七欄位）

每檔通過篩選之標的 **MUST** 輸出以下七欄位，順序固定、缺一即不合規：

| # | 欄位 | 規格 |
|---|---|---|
| 1 | 評分 | 整數 0–100（含端點） |
| 2 | 信號 | 買進／持有／觀望／賣出（四選一） |
| 3 | 目標價 | 數值（新台幣元） |
| 4 | 止損價 | 數值（新台幣元） |
| 5 | 核心理由 | 條列 1–3 點 |
| 6 | 風險提示 | 非空文字 |
| 7 | data_gaps | 陣列，如實反映模組缺口，不得省略或偽裝為空 |

## 7. 評分權重與指標

綜合評分 = **基本面 30% × 技術面 30% × 籌碼面 40%**。權重與各面向指標定義（方向語意、回看窗口、資料來源）以 [`references/scoring-model.md`](references/scoring-model.md) 為**單一真相來源**——調權重只改該檔、不動本流程。任一面向因 `data_gaps` 缺資料時，依 scoring-model.md **重新正規化**剩餘面向，**不以預設值填充**。

**策略 preset**：使用者指名策略（如「用高股息策略挑…」）時，套用 [`references/strategy-presets.md`](references/strategy-presets.md) 該 preset 的**面向權重覆寫**與強調指標；未指名則用預設 30/30/40。指標算法仍以 scoring-model.md 為準。

## 8. 目標價、止損價與風險 overlay

- **價位以市場價（新台幣元）呈現**，供**觸價判定**；**交易成本不扣進價位**。v1.0 **long-only**（信號賣出／觀望 = 不建議買進／退出，非做空）下滿足 **止損價 ≤ 現價 ≤ 目標價**。
- **預期報酬率**計算 **MUST 扣除來回成本**（買 0.1425% + 賣 0.4425% ≈ 0.585%），引用 `references/tw-market-rules.md`，禁止另編成本。
- **乖離率 overlay**：正乖離過大時於**風險提示**欄標「追高留意回檔」（依 scoring-model.md，不計入評分）。
- 標的觸及漲跌停（±10%）或列入 TWSE 處置名單 → 風險提示欄標註**流動性風險**。

## 9. 合規（紅線）

- **data_gaps 非空** → 看板 **MUST 明示**「以下判讀基於不完整資料」；為空則不加此警語。
- **每檔標的必附免責聲明**（不論 data_gaps 是否為空）。
- **禁止**「保證獲利／必漲／穩賺」等表述；評分、信號、目標價僅為**參考資訊**，不宣稱達成為必然。

## 10. 範例（決策看板）

```
【2330 台積電】
評分：78 / 100
信號：買進
目標價：1,050 元　止損價：920 元（市場價，供觸價判定；成本於預期報酬率計算，見 scoring-model.md）
核心理由：
  1. 外資連續 5 日買超
  2. 月營收 YoY +22%
  3. 站上季線、量增價揚
風險提示：接近前高，追高留意回檔；大盤系統性風險。
data_gaps：[]
⚠️ 免責：本分析僅供研究參考，不構成投資建議，投資決策請自行判斷並自負風險。
```

data_gaps 非空時，看板頂端加註：「以下判讀基於不完整資料」。

## 11. 分層邊界與免責

- 本場景為 **L3 企業場景**：只做編排、篩選、看板呈現；**取數一律委派 L2 `twstock-module`**。
- 所有輸出**不構成投資建議**。台股有漲跌停與流動性風險，AI 分析存在不確定性，投資決策請自行判斷並諮詢專業人士。
